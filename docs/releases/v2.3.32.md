# VCO Release v2.3.32

- Date: 2026-03-13
- Commit(base): fa293a7

## Highlights

- Added a governed full-setup install path for both Windows and Linux / macOS.
- Added `scripts/bootstrap/one-shot-setup.sh` so shell users can install the shipped runtime payload, materialize the active MCP profile, and finish with a deep readiness check.
- Updated `README.md`, `README.en.md`, `docs/one-shot-setup.md`, and `docs/deployment.md` so operators can distinguish between:
  - what the repo installs automatically
  - what still requires host-managed plugin / MCP / secret provisioning
- Kept canonical and bundled mirrors in sync and refreshed `config/skills-lock.json` to match the new shipped surface.

## Migration Notes

- Linux / macOS users should now prefer `bash ./scripts/bootstrap/one-shot-setup.sh`.
- Windows users should continue to use `scripts/bootstrap/one-shot-setup.ps1`, with Windows PowerShell fallback when `pwsh` is unavailable.
- A deep check ending in `manual_actions_pending` is expected until host plugins (`superpowers`, `everything-claude-code`, `claude-code-settings`, `hookify`, `ralph-loop`), plugin-backed MCP surfaces (`github`, `context7`, `serena`), and required provider secrets are provisioned.
